## 10. Аналіз системи безпеки

Забезпечення інформаційної безпеки є критичною вимогою для корпоративних систем. Захист системи будується на кількох рівнях. На мережевому рівні весь обмін даними між клієнтом і сервером здійснюється виключно за зашифрованим протоколом HTTPS з використанням сучасних алгоритмів TLS, що унеможливлює перехоплення інформації про вантажі (атака Man-in-the-Middle).

На рівні додатку реалізовано надійний механізм автентифікації. Паролі користувачів не зберігаються у відкритому вигляді; натомість база даних містить лише їх криптографічні хеші (генеруються алгоритмом bcrypt із використанням "солі"). Для управління сесіями застосовується технологія JSON Web Tokens (JWT). Токени мають обмежений час дії, а всі запити до API проходять перевірку на наявність валідного токена та відповідність ролі користувача (авторизація).

Для захисту від вразливостей пристрою бази даних (зокрема SQL ін'єкцій) архітектура передбачає використання ORM-фреймворків або параметризованих запитів, які автоматично екранують вхідні дані від користувачів. Цілісність і доступність даних забезпечується політикою регулярного резервного копіювання з автоматичним збереженням дампів бази у віддалене хмарне сховище.
