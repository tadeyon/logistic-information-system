## 2. Збір і аналіз вимог (Специфікація SRS)

### 2.1. Бізнес-процеси
1. **Реєстрація транспортного засобу:** Інформація про вантажівку, водія, документи.
2. **Створення вантажу (Create Load):** Звідки - куди, часові рамки, інформація про сам вантаж (кількість одиниць, вага, non/dangerous good, non/temperature-control), а також закріплення юніта, який виконуватиме перевезення.
3. **Пошук транспорту:** Пошук машин за фільтрами для виконання створеного замовлення.
4. **Призначення вантажу:** Зв'язування сутності "Вантаж" із сутністю "Транспортний засіб".

### 2.2. Функціональні вимоги
* Система повинна дозволяти створення, редагування та видалення записів про автомобілі (CRUD операції).
* Система повинна дозволяти пошук автомобілів за параметрами: `Location (Zipcode + Радіус)`, `Vehicle Type` (Sprinter, Box truck тощо), `Max Weight`, `Cargo Dimensions` (Length, Width, Height).
* Система повинна надавати форму для створення вантажу з полями: назва facility, адреса, zipcode (для pickup та delivery), piece count, total weight, і т.д.

### 2.3. Нефункціональні вимоги
* **Продуктивність:** Час відгуку під час пошуку автомобілів за фільтрами не повинен перевищувати 2 секунди.
* **Доступність:** Система повинна працювати в режимі 24/7 (SLA 99.9%).
* **Інтерфейс:** Веб-орієнтований, адаптивний інтерфейс для зручної роботи з ПК диспетчера.
